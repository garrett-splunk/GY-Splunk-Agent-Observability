INSTRUMENTATION.md · Facilitator guide

Hands-on lab · ~60–90 minutes

Instrument an AI Assistant with Galileo

Build a LangGraph IT Helpdesk agent, wire Galileo SDK observability step by step, and run demo scenarios for hallucinations and prompt injection in Splunk Agent Observability.

Key idea

The app runs before you finish instrumentation. Comment blocks marked # INSTRUMENTATION: are your exercises — uncomment and wire Galileo until traces appear in the console.

Architecture

Streamlit UI
Chat + demo sidebar
invoke
LangGraph Agent
lookup_ticket · search_kb
GalileoCallback → LLM + tool spans trace lifecycle → root span
export
Galileo Console
Traces · evaluators · guardrails

Lab root: ~/Desktop/galileo-assistant-lab · Reference: galileo-golden-demo

Step 1 ~2 min

Prerequisites

  • Python 3.10+
  • Ollama with gemma4 pulled — or OpenAI API key
  • Galileo API key + access to Galileo console
  • ~500 MB disk for Python venv + dependencies
  • Ports 8501 (Streamlit app) and 8095 (this guide, optional) available

Ollama quick check

curl http://localhost:11434/api/tags should return JSON. If empty, run ollama pull gemma4.

Step 2 ~5 min

Set up the lab

Create a virtual environment and install dependencies.

2.1 — Go to the lab folder

terminal 
cd ~/Desktop/galileo-assistant-lab

2.2 — Create venv and install packages

terminal 
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

2.3 — Open this workshop guide (optional)

terminal · second window 
cd ~/Desktop/galileo-assistant-lab/workshop
python3 -m http.server 8095
# browse http://localhost:8095
Step 3 ~5 min

Configure secrets

Copy the example secrets file and add your Galileo and LLM credentials. Never commit secrets.toml.

3.1 — Copy secrets template

terminal 
cp .streamlit/secrets.toml.example .streamlit/secrets.toml

3.2 — Edit secrets

Set at minimum:

  • galileo_api_key — from Galileo console
  • galileo_console_url — your Galileo instance URL
  • openai_api_key — if using Hosted provider (optional if using Ollama)

3.3 — Verify environment (Exercise 1)

When the API key is set, setup_env.py loads GALILEO_PROJECT from config.yaml.

terminal 
source venv/bin/activate
python -c "from setup_env import setup_environment; setup_environment(); import os; print('project:', os.environ.get('GALILEO_PROJECT'))"

Expected: project: galileo-lab-it-helpdesk

Step 4 ~5 min

Run the assistant

Start Streamlit. The app works without instrumentation — traces appear after Steps 5–6.

terminal 
cd ~/Desktop/galileo-assistant-lab
source venv/bin/activate
streamlit run app.py

Open http://localhost:8501

  • Sidebar shows model provider (Local Ollama or Hosted OpenAI)
  • Sidebar Demo Scenarios section is visible
  • Chat responds to “Look up ticket T-1001”
Step 5 ~10 min

Instrument: environment + session logger

Complete Exercises 1–2 in INSTRUMENTATION.md. Search for # INSTRUMENTATION in the repo.

Exercise 1 — setup_env.py

Confirm GALILEO_* env vars are set when API key is present (auto-wired if secrets are configured).

Exercise 2 — app.py

  1. Uncomment from galileo import GalileoLogger
  2. Implement _create_galileo_logger() (see comment block)
  3. Implement _start_galileo_session_if_needed() (see comment block)

Reference: galileo-golden-demo/app.py lines ~304–317 and ~1024–1076

Checkpoint

Restart Streamlit. Sidebar should show Galileo API key loaded. Send a chat message — a new session should appear in Galileo console.

Step 6 ~15 min

Instrument: callback + trace lifecycle

Complete Exercises 3–4. This wires automatic LLM/tool spans and a root trace per chat turn.

Exercise 3 — agent.py

  1. Uncomment GalileoCallback import
  2. Replace self.invoke_config with the commented version including callbacks
  3. Keep start_new_trace=False and flush_on_chain_end=False

Exercise 4 — helpers/trace_lifecycle.py

  1. Uncomment bodies of ensure_trace_started() and finalize_trace()
  2. Calls in agent.py are already in place

Reference: galileo-golden-demo/agent_frameworks/langgraph/agent.py and helpers/agent_control_helpers.py

Checkpoint

Send “Look up ticket T-1001”. In Galileo, open trace Run Agent with nested LLM and lookup_ticket tool spans.

Optional: Exercises 5–6

  • Exercise 5 — Named LLM spans in helpers/llm.py (already set; verify labels)
  • Exercise 6 — Manual add_tool_span in tools.py
Step 7 ~5 min

Demo: happy path

Act 1 — show normal agent observability.

  1. In the app sidebar under Act 1 — Happy path, click Look up ticket T-1001
  2. Wait for the agent response with ticket details
  3. Open Galileo → project galileo-lab-it-helpdesk → latest trace

Point out: LLM planning step, tool execution, final synthesis. Try How do I reset my VPN password? to trigger search_kb.

Step 8 ~5 min

Demo: hallucination

Act 2 — intentional grounding failure for evaluator demos.

  1. Sidebar → Act 2 — Hallucination
  2. Click Log Hallucination to Galileo
  3. Open the new trace in Galileo console

Compare context vs output

Retriever span: P1 SLA — 15 min response, 4 hr resolution
LLM span: 24 hr response, 5 day resolution (wrong)

Edit scenarios in config.yaml → demo_hallucinations. Implementation reference: helpers/demo_scenarios.py.

Step 9 ~5 min

Demo: prompt injection

Act 3 — show malicious input captured in traces.

  1. Sidebar → Act 3 — Prompt injection
  2. Click an example chip (e.g. Ignore instructions)
  3. Review prefilled text in chat — send intentionally
  4. Show trace input in Galileo

Customize payloads in config.yaml → demo_prompt_injections.

Step 10 ~5 min

Verify in Galileo

Complete this checklist in the Galileo console:

  • Project: galileo-lab-it-helpdesk
  • Log stream: default
  • Live agent trace: root → LLM → tool → LLM
  • Hallucination trace: retriever + LLM mismatch
  • Injection trace: full malicious prompt in input

Phase 2 — Live guardrails (optional)

Move from observe to block prompt injection using Agent Control.

  1. Add agent-control-sdk[galileo] to requirements
  2. Wire enable_agent_control() and @control on the LLM step (see golden demo)
  3. Create block-prompt-injection control on the log stream in Galileo UI
  4. Replay Act 3 — show deny/steer evaluation span

Details: galileo-golden-demo/README.md (~lines 470–484) and ../DEMO.md.

Troubleshooting

SymptomLikely fix
No tracesCheck galileo_api_key in secrets; complete Steps 5–6
Duplicate root tracesGalileoCallback(start_new_trace=False)
LLM spans, no tool spansAttach callback in agent.py (Exercise 3)
Ollama errorsollama serve and ollama pull gemma4
Hallucination button failsAPI key or project name in config.yaml
